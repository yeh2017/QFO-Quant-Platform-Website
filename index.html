Module 00

设置与启动：所有模块运行前的第一步

Token / API Key / Runtime

为什么放在最前面

设置模块是 QFO 的运行前置条件。数据同步、新闻检索、LLM 情绪分析、推送通知都依赖这里的密钥和开关；没有完成设置，后续模块即使界面能打开，也无法获得稳定数据和分析结果。

建议操作路径

  1. 先启动后端与前端，确认平台可以正常访问。
  2. 打开设置入口，填入 Tushare Token，这是数据中台的必填项。
  3. 按需填入 Tavily API Key、LLM Key 和推送配置，用于新闻检索、新闻理解与通知。
  4. 保存配置后，再进入数据中台执行同步，最后进入选股、新闻和回测模块。
启动项目填写密钥保存配置同步数据进入研究

Module 01

数据中台：量化研究的地基

Tushare / AkShare / BaoStock

这一模块要解决什么

量化研究的第一性原理很简单：没有稳定数据，就没有可信回测。QFO 的数据中台负责把多源金融数据同步到本地仓库，减少重复请求，并为后续选股、因子、回测提供统一入口。

操作路径

  1. 在设置里填入 Tushare Token。
  2. 选择同步范围：A 股行情与因子、ETF、可转债、财务数据、行业指数。
  3. 执行一键全量同步，等待本地仓库更新完成。
  4. 确认顶部统计卡：股票数、ETF 数、可转债数量和行情更新时间。

关键文件

Module 03

因子模块：给机会一个可解释理由

10 Factor System

核心概念

因子的本质是把股票特征变成可比较的数字。QFO 使用价值、质量、动量、规模、低波、成长、红利、集中度、应计、反转等维度，让选股结果有可复盘依据。

去极值：降低异常值对横截面排名的影响。

标准化：把不同量纲的指标转成可加权比较的分数。

加权合成：把多个因子组合成最终排序信号。

backend/services/factors/calculator.py
def normalize_factor(series):
    clean = winsorize_by_mad(series)
    return (clean - clean.mean()) / clean.std()

def score_factors(factors, weights):
    return sum(factors[name] * weight for name, weight in weights.items())

Module 07

新闻中心：LLM 把新闻转成可研究信号

News / LLM / Event Backtest

LLM 是新闻模块的核心

新闻模块不是简单聚合新闻，而是用 LLM 对新闻进行理解、去噪、归因和情绪判断：识别事件主体，关联股票，判断影响方向与强度，再把结果交给事件统计和回测验证。
